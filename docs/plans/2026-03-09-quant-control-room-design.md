# Quant Control Room Design

## Goal

Transform the current single-market panel into a mixed-mode quant control room that can grow into a full trading system without throwing away the existing market data workspace.

## Experience Direction

The interface should feel like an institutional operations desk rather than a landing page or a demo dashboard. The layout needs three clear zones:

1. A persistent left navigation rail for system modules.
2. A central market workspace that keeps the chart as the primary surface.
3. A right operations rail for strategy, risk, positions, and event monitoring.

## Visual Direction

- Dark, dense, operator-first terminal aesthetic.
- Strong grid, sharp panel edges, restrained accent colors.
- Typography should separate system chrome, market numbers, and analytical labels.
- Decorative treatment should support atmosphere without reducing readability.

## Content Strategy

- Keep real market functionality in the center column.
- Add high-quality shell modules for strategy, risk, positions, and execution.
- Make placeholder modules look intentional and production-ready, not empty boxes.
- Use clear labels such as `Planned`, `Standby`, `No active strategy`, so the shell reads as an evolving product instead of fake data.
