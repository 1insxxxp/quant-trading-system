# 终端化行情页面精修设计

## 目标

把当前行情页从“已有主题壳层和图表功能，但画面仍偏松散的后台页面”收紧成更像专业交易终端的主工作面。重点优化三块：

- 顶栏：更轻、更精致，不再占据过多注意力
- 市场参数带：更贴近图表，减少表单感和空洞感
- 图表主画面：HUD 更克制、层次更稳、加载与空状态更专业

## 当前问题

### 顶栏

- `SystemTopbar` 现在信息结构正确，但视觉重心偏重，实时价格、主题切换、时间和在线状态像四块并列卡片。
- 亮暗主题已经区分，但顶栏仍然保留较强容器感，与“专业终端的一条状态条”有距离。

### 市场参数带

- `Toolbar` 的功能齐全，但交易所 / 交易对 / 周期仍然像独立筛选表单。
- 选择区与图表头关系不够紧，视觉上有明显空档，导致主图区域缺少“工作面一体化”感觉。

### 图表工作区

- `ChartInspector` 现在比早期大卡片版已经轻很多，但 HUD 仍偏厚，信息层级不够扁平。
- 图表 body 的包裹层和状态遮罩还可以更克制，当前 loading / empty state 仍像“页面提示”，不像“终端状态”。
- 中文文案仍有个别历史编码遗留，需要顺手统一清掉。

## 设计方向

采用“专业终端收紧版”，而不是继续叠加中台卡片感。核心原则：

- 图表是绝对主角，其他元素只服务图表
- 控件和状态尽量做薄，避免厚卡片和大面积留白
- 暗系保持青绿深海调，亮系保持白蓝科技调
- 用细描边、局部光感和数字排版，而不是重背景块去建立层级

## 顶栏设计

### 结构

- 左侧：侧边栏切换按钮 + 实时价格条
- 实时价格条内只保留：
  - 最新价格
  - 交易对
  - 交易所
- 右侧：主题切换、系统时间、在线状态，做成一条更紧凑的终端状态带

### 视觉

- 顶栏高度继续收紧
- 去掉过多胶囊边界感，改成一条更细、更平的状态容器
- 最新价格用 `tabular-nums`
- 在线状态保留单点信号灯，但容器感更弱

### 可访问性

- 侧边栏按钮和主题切换按钮保留清晰 `aria-label`
- 焦点态使用 `:focus-visible`
- 系统时间继续使用 `Intl.DateTimeFormat`

## 市场参数带设计

### 结构

- 把交易所 / 交易对 / 周期重构为一条贴图表头的“参数工具带”
- 左侧优先放交易所与交易对
- 中间放周期
- 右侧放指标入口和极少量运行态提示

### 交互

- 选择器高度缩短
- 减少独立大输入框感，强调“终端参数切换”
- 选中态依赖描边、轻底色和图标，不使用厚重底块

### 可访问性

- 自定义选择器继续保留键盘导航
- 标签文本和控件层级更清晰，避免仅靠占位文案表达语义

## 图表与 HUD 设计

### HUD

- HUD 改成真正的图内信息层，不再像悬浮卡片
- 分为两层：
  - 主层：交易对 / 周期 / 交易所 / 当前 candle 时间
  - 次层：开高低收 / 涨跌 / 成交量 / 指标
- 颜色只作用于关键数值，不让整块 HUD 变成强视觉主体

### 图表主体

- 降低 `chart-panel__body-frame` 和外围装饰感
- 让画布边界更隐，主画面更完整
- 成交量说明、指标 legend 和 HUD 之间建立更稳定的垂直节奏

### 加载与空状态

- loading 状态从大面积说明遮罩改成更轻的终端状态提示
- empty state 只保留简洁文案和状态语气，不再像“页面引导卡片”
- 历史加载中的状态条继续保留，但位置、尺寸和颜色再收敛

## 主题策略

### 暗色主题

- 主背景：深海蓝黑
- 功能强调：青绿、冷蓝
- 状态成功：偏青绿
- 危险/下跌：保留柔和红，不做刺眼高饱和

### 亮色主题

- 主背景：白蓝科技灰
- 强调色：清冷蓝
- 局部高光比暗色更轻，不做过多磨砂块

## 非目标

- 不改后端接口
- 不改 WebSocket 协议
- 不新增交易功能
- 不重做图表库和数据链路

## 验证标准

- 顶栏首屏视觉高度明显降低
- 市场参数带更贴近图表、空洞感减少
- HUD 视觉重量显著降低，但信息读取更快
- loading 和 empty state 更像终端状态，而不是大说明层
- 亮暗主题都保持统一语言
- 相关组件测试与前端构建继续通过
