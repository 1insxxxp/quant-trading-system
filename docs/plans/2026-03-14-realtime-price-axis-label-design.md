# 实时价格轴标签重构设计

**日期：** 2026-03-14

## 背景

当前 K 线图右侧“实时价格 + 倒计时”使用自绘 DOM 浮层实现。这个实现有三个问题：

- 价格块会超出价格轴可视边界
- 价格字号与右侧坐标轴刻度字号不一致
- 自绘价格块和图表原生价格轴标签并存，视觉语义冲突

用户要求把实时价格改成图表价格轴自己的标签，同时保留倒计时，并且整体不越界。

## 目标

- 实时价格使用 `lightweight-charts` 原生价格轴标签
- 倒计时保留为第二行信息，显示在价格轴内部
- 价格字号与右侧坐标轴刻度字号保持一致
- 去掉原先自绘大色块与多余侧边框

## 方案

### 价格

不再把价格文本画在 `chart-realtime-badge` 中，而是将实时价格线 `PriceLine` 的：

- `axisLabelVisible` 设为 `true`
- `lineVisible` 保持 `true`
- `price` 继续跟随最新价格

这样价格本身由图表价格轴原生渲染，字号、宽度和定位完全跟随图表库自身逻辑。

### 倒计时

保留一个极简的自绘倒计时 DOM，但仅显示倒计时，不再重复显示价格：

- 位置锚定到当前价格坐标下方
- 限制在右侧价格轴区域内部
- 字号比价格小一档
- 背景与边框全部移除，避免与原生价格轴标签争抢视觉层级

### 视觉约束

- 价格由图表价格轴原生标签渲染
- 倒计时仅作为第二行辅助信息
- 价格与倒计时保持垂直两行关系
- 不允许任何元素超出右侧价格轴边界

## 涉及文件

- `frontend/src/components/KlineChart.tsx`
- `frontend/src/index.css`
- `frontend/src/index.css.test.js`

- 刷新并切换周期后，右侧只出现图表原生价格轴价格标签
- 倒计时显示在价格标签下方，不越界
- 价格字体大小与右侧坐标轴刻度一致
- `npm test -- --run src/index.css.test.js`
- `npm run build`
